Hadoop

A History

In order to know how we got to where we are today, we should take a brief look back at how the Hadoop ecosystem was planted and grew.
Most think of Hadoop and this Yellow Elephant pops into their head.
The next thought usually goes to "something something big databases" however Hadoop is much more than a solution to the limitations of traditional Relational Databases.

1997-98

Lucene

Scalable Search

What powers Google Searches

2001

Nutch

Crawl the whole Internet

How hard could it be?

The Hadoop ecosystem can trace its roots back to the early days of the Internet and everything we know and use today.
In 1997, Doug Cutting started working on the first version of what would become Apache Lucene, a full-text search library. It's what powers the sub-second query results that you expect from Google. Great, with fast search, we can now inspect data.
With the help of Mike Cafarella, Doug created what is now Apache Nutch, a crawler, spider, or otherwise a data-scraping robot. They started to literally crawl the entire Internet, and with Lucene were able to search across everything crawled almost instantly.
As you may assume, crawling and searching the entire Internet is a larger order.
It was around 2002 where they started to experience scalability issues and started to design a system in software that was schemaless, durable and self-healing, and could be deployed on commodity hardware.

2003

GFS

Google File System

Scalable FS

2004

NDFS

Nutch Distributed File System

GFS, but in Java!

Around this time, Google releases their specification for Google File System which solved many of the issues Doug and Mike were facing so they set to implement it in Java and thus the Nutch Distributed File System was born.
Now that these projects could tackle the task of crawling, searching, and storing large quantities of data, there was the challenge of processing this data in a distributed fashion. Most other solutions would transport the data to a large computational engine and then ship the results out. At this sort of scale that was unfeasible.

2005

MapReduce

Integrated with Nutch

2007

Yahoo! Hadoop

1000-nodes in Production

2006

Hadoop was born

Enter MapReduce, by Jeffery Dean and Sanjay Ghemawat (Gimma-watt). MapReduce solved parallelization, distribution, and fault-tollerance by Mapping parts of a job to all nodes, and then Reducing slices of the work back down to the final results.
Around this time, the different projects were moved from sub-projects of the Lucene organization, into a new Hadoop ecosystem which consisted of Hadoop Common, HDFS, and MapReduce.
Hadoop started to pick up steam, enough to where Yahoo! (if anyone remembers them) replaced their legacy C++ based search engine with Hadoop and within 6 months were running a 1000-node cluster in production. How could a Java program replace a robust C++ system? Surprisingly it did so with relative ease.

2008

Ecosystem Explosion

2008

ASF Induction

2008

Between the community and Hadoop's use in production, Hadoop was graduated to a top-level Apache Software Foundation project.
After this, we saw an explosion of other projects submitted to be part of the Hadoop ecosystem
  • HBase offered a natural database on top of the scalable HDFS
  • Zookeeper aided in coordinating the distributed systems
  • Yahoo contributed their Pig programming language that extended MapReduce
  • Facebook contributed Hive, the first incarnation of SQL on top of MapReduce
  • And if that weren't enought, in 2008 Cloudera was founded by Hadoop experts from BerkleyDB, Google, Facebook, and Yahoo!

2009


There are too many events in the history of the Hadoop ecosystem to chart through all of them at this point, but a few to make mention are:
  • By March 2009, Amazon was already offering MapReduce hosting, Elastic MapReduce.
  • In June of 2011, Hortonworks was founded by 8 Apache Hadoop Committee members from Yahoo!
  • By 2012, Yahoo!'s Hadoop cluster hit over 42,000 nodes, and there are over 1,200 contributors to the Hadoop projects.

Data Integration

B2B, B2C, B2B2C2U

As data drove more and more value and helped move IT from a cost-center to a revenue generating part of the business, integrations became mandatory.
Not simply between different teams or systems in your organization, but interchanges between businesses helped pave the way for the hyper-aware and hyper-active capabilties that now give us the ability to know within moments of when an Amazon package has been delivered.
